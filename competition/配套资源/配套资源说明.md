# 配套资源说明

本目录是参赛案例的"配套资源"打包目录。依据《创AI案例指南》"配套资源"要求整理。

## 目录结构

```
配套资源/
├── README.md                    # 项目主 README（含介绍、特性、快速开始）
├── LICENSE                      # Apache 2.0 开源协议
├── 配套资源说明.md              # 本文件
├── 文档/
│   ├── 安装手册.md              # 完整安装部署指南
│   ├── 使用手册.md              # 详细功能使用说明
│   ├── 开发记录.md              # 开发历程与AI使用情况
│   └── AI生成内容声明.md        # AI 生成内容标记说明
├── images/                      # 系统架构图、流程图、功能图
│   ├── 01_system_architecture.png
│   ├── 02_feature_modules.png
│   ├── 03_quality_flow.png
│   ├── 04_tech_stack.png
│   └── 05_rag_flow.png
└── 源代码/                      # （需要你手动放入完整 backend/ 与 frontend/ 目录）
    ├── backend/
    └── frontend/
```

## 提交前打包步骤

1. 将项目根目录的 `backend/` 和 `frontend/` 复制到本目录的 `源代码/` 下
2. 注意排除以下大文件/敏感文件：
   - `backend/target/` — Maven 构建产物
   - `backend/data/` — 用户上传文件、生成的 Word/Excel
   - `backend/logs/` — 运行日志
   - `frontend/node_modules/` — npm 依赖
   - `frontend/dist/` — 前端构建产物
   - 任何 `.env` 或包含 API Key 的文件
3. 检查 `backend/src/main/resources/application.yml` 中 API Key 是否已清空（不要把你的 API Key 上传）
4. 将整个 `配套资源/` 目录压缩为 ZIP（推荐格式：UTF-8 文件名）
